p2 Storage Engine

Blistering-fast, S3-compatible object storage built on Python + Rust

Petabyte-scale metadata via LMDB. Thousands of objects per second. Zero-copy reads. No framework overhead.

Key Features

🔌

S3 API Compatible

Drop-in replacement for S3. Works with AWS SDKs, MinIO warp, Cyberduck, and any standard REST tooling.

⚡

Microsecond Cryptography

MD5 and SHA256 hashing offloaded to p2_s3_crypto, a custom Rust extension that releases the Python GIL.

📡

Zero-Copy Reads

GET operations use Nginx X-Accel-Redirect to serve files via kernel-level sendfile(), bypassing Python entirely.

🚀

Framework-less Writes

PUT traffic is intercepted at the raw ASGI layer before Django loads — matching compiled Rust backend performance.

🗄️

LMDB Metadata

Petabyte-scale metadata stored in LMDB with lock-free reads, batched writes, and zero-copy memory-mapped access.

🎛️

Control Plane UI

Modern management interface with Django Ninja, fully typed OpenAPI docs, bucket policies, CORS, ACLs, and lifecycle rules.

Data Plane

Raw ASGI + Rust Extensions

  • ASGI interceptor steals raw bytes off Granian sockets
  • Bypasses Django middleware completely
  • p2_s3_crypto computes hashes without stalling the event loop
  • Payload streamed directly to disk
  • Metadata pushed to LMDB via batched async queue
  • Nginx sendfile() for zero-copy GET responses

Getting Started

🐧 Native Linux

bash scripts/run_without_docker.sh
  • Auto-installs dependencies
  • Binds Granian to localhost:8787
  • Generates Nginx config from deploy/nginx-host.conf
  • Set P2_DEBUG=true in .env for verbose logs
  • ~586 MiB RAM across 4 Granian workers at idle

🐳 Docker

docker compose up -d --build
  • Web server on localhost:8787
  • Includes Redis broker + arq workers
  • Storage auto-mounted at ./storage/

Nginx Zero-Copy Setup

For maximum GET performance, pair with Nginx:

# .env
P2_STORAGE__USE_X_ACCEL_REDIRECT=true

Point Nginx at http://127.0.0.1:8787 (Granian). The run_without_docker.sh script generates the config automatically.

Use with AWS CLI

aws configure set default.s3.signature_version s3v4
aws --endpoint-url http://localhost s3 mb s3://my-bucket
aws --endpoint-url http://localhost s3 cp file.txt s3://my-bucket/
aws --endpoint-url http://localhost s3 ls s3://my-bucket/
aws --endpoint-url http://localhost s3 presign s3://my-bucket/file.txt

Unit Tests (35)

  • Presigned Tokens — roundtrip, wrong bucket/key/method, expired, tampered signature, malformed, max expiry clamping
  • CORS — origin matching, wildcard/subdomain, rule matching, header application, XML parse/build roundtrip
  • Bucket Policy Engine — parse valid/invalid JSON, allow/deny evaluation, deny-overrides-allow, wildcard actions, resource matching
  • AWS SigV4 Auth — query string parsing, header parsing, missing params, HMAC key derivation, signing key determinism
  • Error Classes — HTTP status codes and S3 error code strings

Integration Tests (22)

  • Bucket Ops — head, delete, versioning, location, list empty, ACL get/put, CORS put/get, policy CRUD, lifecycle CRUD, invalid policy rejection
  • Object Lifecycle — PUT with ETag/MD5 verification, HEAD (content-type, size, etag), copy object, list after put, presigned URL generation, delete
  • Object Not Found — head 404, get 404, no-such-bucket, tagging 404, ACL 404, copy missing source, multi-delete nonexistent
  • Authentication — invalid credentials rejected, wrong secret key → SignatureDoesNotMatch, multi-user isolation (cross-bucket access denied)
